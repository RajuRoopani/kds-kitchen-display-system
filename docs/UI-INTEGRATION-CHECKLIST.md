Store Subscription\n\n```typescript\n// Custom hook subscribes to store changes\nexport function useOrdersByStatus(status: OrderStatus): Order[] {\n  const [orders, setOrders] = useState<Order[]>(() =>\n    globalOrderStore?.getOrdersByStatus(status) || []\n  );\n\n  useEffect(() => {\n    // Subscribe to all store changes\n    const unsubscribe = globalOrderStore.subscribe((store) => {\n      // Update local state on ANY store change\n      setOrders(store.getOrdersByStatus(status));\n    });\n\n    return unsubscribe;\n  }, [status]);\n\n  return orders;\n}\n```\n\n### Pattern 2: Optimistic UI + Error Recovery\n\n```typescript\n// In useOrderAction hook:\n// 1. Update UI immediately (optimistic)\nstore.updateOrderStatus(orderId, nextStatus);\n\n// 2. Wait for backend confirmation\nawait dispatcher.dispatch(orderId, action);\n\n// 3. On error, revert to previous status\nif (error) {\n  store.updateOrderStatus(orderId, previousStatus);\n}\n```\n\n### Pattern 3: Event Listener Registration\n\n```typescript\n// In App.tsx:\nconst unsubConnected = ws.onConnected(() => setIsConnected(true));\nconst unsubDisconnected = ws.onDisconnected(() => setIsConnected(false));\nconst unsubMessage = ws.onMessage((msg) => handleMessage(msg));\nconst unsubError = ws.onError((error) => handleError(error));\n\n// On unmount:\nreturn () => {\n  unsubConnected();\n  unsubDisconnected();\n  unsubMessage();\n  unsubError();\n};\n```\n\n---\n\n## Performance Considerations\n\n1. **Store Subscription**: Updates trigger on every store mutation\n   - Use selectors (getOrdersByStatus) to minimize re-renders\n   - Components only subscribe to data they need\n\n2. **Re-render Optimization**: Components re-render only when their data changes\n   - useOrdersByStatus only updates when orders of that status change\n   - useOrderMetrics only updates when metrics change\n\n3. **Memory Management**:\n   - Unsubscribe callbacks properly on unmount\n   - Cancel dismiss timers on STATE_SYNC\n   - Clear event listeners on destroy\n\n4. **Message Handling**: All incoming messages processed synchronously\n   - No blocking operations in message handlers\n   - Store mutations are fast (Map operations)\n\n---\n\n## Browser Compatibility\n\n- ✅ WebSocket support: All modern browsers (IE 10+)\n- ✅ CSS Grid: Desktop layout uses CSS Grid (IE 11+)\n- ✅ Flexbox: Mobile layout uses Flexbox (all modern browsers)\n- ✅ ES2020: TypeScript compiled to ES2020\n\n---\n\n## Known Limitations\n\n1. **Offline Mode**: No local persistence\n   - Orders cleared on disconnect\n   - Recovered via STATE_SYNC on reconnect\n\n2. **Action Deduplication**: 500ms window\n   - Rapid identical actions within 500ms are deduplicated\n   - User can retry after 500ms\n\n3. **Dismiss Timer**: Fixed durations\n   - Completed orders: 5 seconds\n   - Cancelled orders: 10 seconds\n   - Cannot be customized per order\n\n4. **Message Parsing**: No partial JSON recovery\n   - Malformed messages are discarded\n   - No buffering or retry logic\n\n---\n\n## Testing Checklist\n\n### Manual Browser Testing\n\n- [ ] **Desktop (1280px+)**\n  -
