# KDS Phase 2 Test Suite\n\n## Overview\n\nThis directory contains comprehensive test coverage for the Kitchen Display System (KDS) Phase 2 implementation, including:\n\n- **Unit tests** for individual components and utilities\n- **Integration tests** for cross-component data flow\n- **Responsive design tests** for mobile/desktop layouts\n- **Accessibility tests** for keyboard navigation and ARIA compliance\n\n## Test Files\n\n### Integration Tests (`integration.test.tsx`)\n\nComprehensive integration test suite covering:\n\n1. **WebSocket Client ↔ OrderStore**\n   - Connection lifecycle (connect, disconnect, reconnect)\n   - Message parsing (ORDER_NEW, ORDER_UPDATE, STATE_SYNC)\n   - Error handling (malformed messages, connection loss)\n   - Event emission (onConnected, onDisconnected)\n\n2. **OrderStore State Management**\n   - Order CRUD operations (upsert, update, remove, replace)\n   - Status filtering (`getOrdersByStatus`)\n   - Metrics calculation (totals, wait times)\n   - Auto-dismiss timers for Completed/Cancelled orders\n   - Subscriber notifications (pub/sub pattern)\n\n3. **Kanban Component Integration**\n   - Rendering with initial store state\n   - Real-time updates when store changes\n   - Connection status display (error banner)\n   - Desktop layout (5-column)\n   - Mobile layout (tab selector + 1 column)\n\n4. **OrderCard Component**\n   - Order data rendering (ID, customer, items)\n   - Accessibility compliance (ARIA roles)\n   - Item list display with quantities\n\n5. **Responsive Layout & Tab Selector**\n   - Tab filtering by order status\n   - Window resize handling (768px breakpoint)\n   - Mobile-to-desktop transition\n   - Keyboard navigation (arrow keys, Enter/Space)\n\n6. **Error Scenarios**\n   - WebSocket connection failures\n   - Rapid message processing\n   - Missing order handling\n   - Store stability under stress\n\n7. **Mock Backend Integration**\n   - Message schema validation\n   - Order structure compliance\n   - Status enum validation\n\n8. **End-to-End Data Flow**\n   - WebSocket → Store → Component → UI\n   - Order lifecycle (NEW → UPDATE → SYNC)\n   - Status transitions\n\n### Other Test Files\n\n- **`Kanban.test.ts`** — Component unit tests (layout, props, responsiveness)\n- **`responsive.test.tsx`** — Responsive design breakpoints\n- **`mobile-integration.test.tsx`** — Mobile-specific integration tests\n- **`OrderCard.test.ts`** — Card component unit tests\n- **`ws-client.integration.test.ts`** — WebSocket client integration\n- **`order-store.test.ts`** — Store unit tests\n- **`accessibility.test.ts`** — ARIA compliance and keyboard navigation\n\n## Running Tests\n\n### Run All Tests\n\n```bash\nnpm test\n```\n\n### Run Specific Test File\n\n```bash\nnpm test integration.test.tsx\n```\n\n### Run in Watch Mode\n\n```bash\nnpm test -- --watch\n```\n\n### Run with UI (Vitest UI)\n\n```bash\nnpm run test:ui\n```\n\n### Run with Coverage\n\n```bash\nnpm run test:coverage\n```\n\n## Test Architecture\n\n### Mocking Strategy\n\n- **WebSocket**: Replaced with `MockWebSocket` class that simulates async connection and message events\n- **Hooks**: Mocked `useOrdersByStatus` and `useOrderMetrics` to use test store\n- **Store**: Real `OrderStore` instance for realistic integration testing\n\n### Test Fixtures\n\nCommon test data creators:\n\n```typescript\n// Create a mock order with defaults\nconst order = createMockOrder({\n  orderId: 'order-123',\n  customerName: 'John Doe',\n  status: 'Received',\n});\n\n// Create mock WebSocket\nconst ws = createMockWebSocket();\n\n// Simulate backend messages\nsimulateOrderNew(ws, order);\nsimulateOrderUpdate(ws, 'order-123', 'Preparing');\nsimulateStateSync(ws, [order1, order2]);\n```\n\n## Coverage Goals\n\n- **80%+ coverage** of integration paths\n- **100% coverage** of critical data flows (WebSocket → Store → UI)\n- **All acceptance criteria** from Phase 2 Inspect with Browser DevTools\n\n```bash\nnode --inspect-brk node_modules/vitest/vitest.mjs run\n```\n\n## Common Issues\n\n### WebSocket Tests Timeout\n\nEnsure `MockWebSocket` calls `open` event after 10ms:\n\n```typescript\nsetTimeout(() => {\n  this.readyState = 1; // OPEN\n  this.dispatchEvent('open');\n}, 10);\n```\n\n### Store Subscription Not Firing\n\nAlways call `store.subscribe()` BEFORE mutating state:\n\n```typescript\nconst listener = () => { /* ... */ };\nstore.subscribe(listener);  // Register first\nstore.upsertOrder(order);   // Then mutate\n```\n\n### Responsive Tests Failing\n\nAlways set `window.innerWidth` before rendering:\n\n```typescript\nObject.defineProperty(window, 'innerWidth', {\n  writable: true,\n  configurable: true,\n  value: 375, // Mobile\n});\n```\n\n## Adding New Tests\n\n1. **Identify the scope**: Unit, integration, or E2E?\n2. **Follow naming**: Test file should match component (e.g., `NewComponent.test.tsx`)\n3. **Use existing fixtures**: Leverage `createMockOrder()`, `createMockWebSocket()`\n4. **Add to acceptance criteria**: Map test to AC if applicable\n5. **Document**: Add summary comment above `describe` block\n\n## Future Improvements\n\n- [ ] E2E tests with Cypress (full browser automation)\n- [ ] Visual regression tests (screenshot comparisons)\n- [ ] Performance benchmarks (render time, message
